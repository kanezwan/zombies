## 一、项目概述

### 1.1 项目目标
模仿经典游戏《植物大战僵尸》（Plants vs. Zombies），开发一款运行在 Android 平台的塔防类休闲游戏。玩家通过种植不同功能的植物，抵御一波波入侵的僵尸，保护房屋不被攻陷。

### 1.2 项目分期

| 阶段 | 目标 | 交付内容 |
| --- | --- | --- |
| **第一期（MVP）** | 完成可玩的最小版本 | 2 种植物 + 3 种僵尸 + 1 张地图 + 单机存档 |
| **第二期（扩展）** | 丰富游戏内容 | 更多植物、更多僵尸、更多关卡、特殊道具、成就系统等 |

### 1.3 核心玩法
- 玩家通过收集"阳光"作为种植资源
- 在 5 行 × 9 列的草坪上选择格子种植植物
- 僵尸从右侧出现，沿行向左推进
- 植物攻击僵尸；僵尸啃食植物
- 任一僵尸越过最左侧 → 游戏失败
- 击退当前关卡所有僵尸波次 → 通关胜利

## 二、第一期：MVP 需求详细说明

### 2.1 功能范围（In Scope）

#### 2.1.1 植物（共 2 种）

| 植物 | 价格（阳光） | 冷却时间 | 功能描述 | 生命值 |
| --- | --- | --- | --- | --- |
| **向日葵** (Sunflower) | 50 | 7.5s | 每隔 24s 产生 25 阳光 | 300 |
| **豌豆射手** (Peashooter) | 100 | 7.5s | 每隔 1.5s 向同行射出豌豆，伤害 20 | 300 |

#### 2.1.2 僵尸（共 3 种）

| 僵尸 | 出现波次 | 移动速度 | 攻击力 | 生命值 | 特殊能力 |
| --- | --- | --- | --- | --- | --- |
| **普通僵尸** (Normal Zombie) | 全程 | 慢 (≈0.4 格/s) | 100/s 啃食 | 200 | 无 |
| **铁桶僵尸** (Buckethead Zombie) | 第 5 波后 | 慢 | 100/s 啃食 | 200 + 铁桶 1100 | 头盔抗性，掉落后变普通僵尸 |
| **读报僵尸** (Newspaper Zombie) | 第 3 波后 | 慢 → 报纸破损后变快 | 100/s 啃食 | 200 + 报纸 150 | 报纸被打掉后激怒，速度翻倍 |

#### 2.1.3 关卡设计
- **唯一地图**：白天-前院草坪（5 行 × 9 列）
- **波次配置**：共 10 波僵尸
  - 第 1-2 波：仅普通僵尸（每波 1-2 只）
  - 第 3-4 波：普通 + 读报僵尸
  - 第 5-9 波：三种混合
  - 第 10 波（Boss 波/最终波）：大量混合僵尸
- **阳光机制**：
  - 自然掉落：每 10s 随机一颗（25 阳光）
  - 向日葵生产：见上表

#### 2.1.4 主要界面（UI）
1. **启动页**：游戏 Logo，1.5s 自动跳转主菜单
2. **主菜单**：开始游戏 / 继续游戏 / 设置 / 退出
3. **战斗页**：
   - 顶部：阳光数量 + 植物卡牌槽（2 张）+ 铲子
   - 中部：5×9 草坪
   - 底部：进度条（提示僵尸波次进度）
4. **结算页**：胜利 / 失败弹窗 → 重玩 / 返回主菜单
5. **设置页**：音量、音效、振动开关

#### 2.1.5 单机存档
- 本地保存当前关卡进度（仅 1 关，可记录是否通关）
- 设置项持久化
- 存储方式：`SharedPreferences` + 单文件 JSON

### 2.2 非功能需求

| 项 | 要求 |
| --- | --- |
| 平台 | Android 7.0 (API 24) 及以上 |
| 屏幕方向 | 横屏 (Landscape) |
| 帧率 | 60 FPS |
| 安装包大小 | < 30 MB |
| 离线 | 100% 单机，无网络依赖 |
| 多语言 | 仅简体中文 |
| 权限 | 仅需存储权限（写存档） |

### 2.3 不在第一期范围（Out of Scope）
- 多关卡/多场景（夜晚、泳池、屋顶等）
- 商店/解锁系统
- 小游戏（保龄球、坚果保龄等）
- 联网/排行榜
- 音乐播放器（保留接口，后期补充）
- 成就系统

## 三、技术方案

### 3.1 技术选型

| 模块 | 选型 | 理由 |
| --- | --- | --- |
| 开发语言 | **Kotlin** | Android 官方推荐，简洁安全 |
| 渲染 | **Android SurfaceView + Canvas** | MVP 简单可控，无需引入第三方引擎 |
| 架构 | **MVVM + ECS 思想** | UI 层用 ViewModel；游戏逻辑用 ECS（Entity-Component-System） |
| 资源管理 | `AssetManager` + 内存缓存 | |
| 持久化 | `SharedPreferences` + JSON | |
| 构建 | **Gradle (Kotlin DSL)** | |
| 最低 SDK | 24（Android 7.0） | 覆盖 95%+ 设备 |
| 目标 SDK | 34（Android 14） | |

> 备注：若后续需要更复杂的物理/粒子，可平滑迁移到 **LibGDX** 或 **Godot**。

### 3.2 模块划分

```
com.zombies/
├── app/                  # 应用入口、Activity
├── ui/                   # 主菜单、设置、结算页等 UI
│   ├── menu/
│   ├── settings/
│   └── game/             # 战斗页 SurfaceView 容器
├── game/
│   ├── core/             # 游戏循环、时间步进、调度器
│   ├── ecs/              # Entity / Component / System 基类
│   ├── entity/           # 植物、僵尸、子弹、阳光实体
│   ├── system/           # 渲染/移动/碰撞/AI/生产系统
│   ├── level/            # 关卡数据、波次配置
│   └── input/            # 触摸输入处理
├── data/                 # 存档、配置
├── resource/             # 资源加载封装
└── util/                 # 工具类
```

### 3.3 核心数据结构（伪代码）

```kotlin
// 实体基类
abstract class Entity(val id: Long) {
    val components = mutableMapOf<KClass<*>, Component>()
}

// 组件示例
data class PositionComponent(var row: Int, var col: Int, var pxX: Float, var pxY: Float) : Component
data class HealthComponent(var hp: Int, val maxHp: Int) : Component
data class AttackComponent(val damage: Int, val intervalMs: Long, var lastFireMs: Long) : Component
data class MoveComponent(var speedPxPerSec: Float) : Component

// 系统示例
class MovementSystem : System { override fun update(dtMs: Long) { /* ... */ } }
class CollisionSystem : System { /* 子弹 vs 僵尸；僵尸 vs 植物 */ }
class SpawnSystem    : System { /* 按波次配置生成僵尸 */ }
class SunSystem      : System { /* 自然掉落 + 向日葵生产 */ }
class RenderSystem   : System { /* Canvas 绘制 */ }
```

### 3.4 游戏循环
- 主线程：UI 事件
- 渲染线程：固定 60 FPS，逻辑步长 16.6ms
- 使用 `Choreographer` 或独立 `HandlerThread` 驱动 SurfaceView 双缓冲

### 3.5 资源规划（占位 → 美术替换）
- 植物贴图：每种植物 1-3 帧动画（PNG，128×128）
- 僵尸贴图：行走 4-6 帧、啃食 2-3 帧、死亡 2-3 帧
- 草坪背景：单张大图（建议 1920×1080）
- 阳光、子弹：小图标
- 音效：种植、射击、阳光收集、僵尸出场、失败音乐（可选 OGG）

### 4.3 风险与对策

| 风险 | 等级 | 对策 |
| --- | --- | --- |
| 美术资源缺失 | 高 | MVP 用免费/占位素材；关键节点引入美术 |
| 自渲染性能不足 | 中 | 监控帧率，必要时切换 LibGDX |
| 多分辨率适配 | 中 | 设计阶段统一虚拟分辨率 + 等比缩放 |
| 触摸交互手感差 | 中 | 增加点击容差、视觉反馈 |
| 进度延期 | 中 | 每周回顾，必要时砍掉非核心特性（如读报僵尸激怒） |

### 4.4 验收标准（DoD - Definition of Done）
1. 在 Android 7.0+ 真机上可流畅运行（≥55 FPS）
2. 完整进行一局：从主菜单 → 战斗 → 通关结算
3. 三种僵尸均按设计行为表现
4. 两种植物均可正常种植与发挥功能
5. 无 Crash、无明显内存泄漏
6. APK 体积 ≤ 30 MB
7. 通关进度可保存并下次进入显示
